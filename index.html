The app

This is the webpage for the application TV1215. TV1215 is an Android application that let’s a person connect their Android phone to a Macbook. The user can use this app to control different applications on the Macbook, as for instance Spotify, Youtube, Netflix and Imgur. The app uses gestures to do the different commands for the different applications.

Overview

This section will descripe our project, with goals, objectives, functionalities, limitations and challenges.

Project goal and objective

For this project we wanted to make a remote control for a Macbook, using an Android application. We wanted to be able to hook the computer up with a TV and then be able to control the computer from the sofa, by only using the Android app. By making such an application, we remove the need for purchasing external hardware, such as a wireless keyboard and mouse. To remove the need for visual attention when using the app, we implemented 9 different gestures that can be used to control different parts of different applications. The communication is done over Bluetooth Low Energy.

Key functionalities

When you open the Android app, you will see a list of application modes. These modes includes information about what each gesture will do. When you select one of the modes, the Macbook will automatically open the corresponding application.

The mentioned gestures allows you to perform application-specific tasks. For instance, when you are using YouTube, you can enter and exit fullscreen, control the volume, play and pause the video and go to the next video, all by using the different gestures.

Every application mode will contain the "Keyboard & Mouse" option. In every application mode, the "Move backwards"-gesture will bring you to the "Keyboard & Mouse"-activity. This activity contains a touch pad that can be used to control the mouse pointer on the Macbook. The touchpad is also clickable just like the touch pad on the computer. To the right of the touch pad, you will find the scroll pad. Moving your finger up and down the scroll pad will cause the current application on the Macbook to scroll. We also have implemented the option of text input. The user can write text with the Android keyboard, and this will then be sent to the Macbook.

Communication

To be as energy aware as possible, we chose to use Bluetooth Low Energy as our communication medium. We set the Android app as a peripheral and wrote an Objective-C OS X application that acted as a client. We have several different bluetooth characteristics for the different actions, and made the OS X app subscribe to the changes in the peripheral's characteristics.

Progress

Here is our progress for our project.

  • Week 6

    Set up projects. Implemented first gesture recognition.

  • Week 7

    Able to activate keyboard keys from the OSX app. Able to detect three different gestures (left and right tilt and forward tilt). Started working on the bluetooth communication.

  • Week 8

    Started looking into the opportunity for using machine learning for gesture recognition. Established Bluetooth connection between phone and macbook. Able to use gestures to controll the macbook.

  • Week 9

    Added support for changing songs and play/pause music on Spotify. Added support for controlling power-point presentations. Added support for browsing Imgur. Implemented six new gestures. Implemented a way to switch between applications on the Macbook

  • Week 10

    Added a mousepad in the Android app that controls the mouse pointer on the mac, both movement and clicks. Also added a scrollpad that enables scrolling on the mac. Implemented a keyboard on the Android app that lets a user write text on the mac.
